# LLM Usage Guide

This page is for AI coaches, agents, and humans preparing context for an LLM.

## Canonical LLM entry points

Use these source files from the repository:

- [llms.txt](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/llms.txt)
- [llms-full.txt](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/llms-full.txt)
- [manifest.json](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/manifest.json)
- [indexes/INDEX_BY_TASK.md](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/indexes/INDEX_BY_TASK.md)
- [indexes/INDEX_BY_CONCEPT.md](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/indexes/INDEX_BY_CONCEPT.md)
- [indexes/INDEX_FOR_AI_COACHES.md](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/indexes/INDEX_FOR_AI_COACHES.md)

## Retrieval discipline

1. Start from `llms.txt` or the smallest relevant index.
2. Route to the smallest useful chapter, topic, shortcut, tool, or field-practice file.
3. Stop at `VCB:STOP_RETRIEVAL` unless the task explicitly requires related topics.
4. Treat volatile product details, pricing, UI labels, model guidance, and exact commands as requiring current official-source confirmation.
5. Treat field-practice cards as candidates unless a later source/evidence review changes their status.

## What the docs site changes

The docs site adds human-friendly navigation. It does not change LLM routing files, source IDs, topic IDs, or stop markers.
