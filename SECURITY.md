# Security Policy

The Vibe Coding Bible is a documentation and knowledge-base repository. Security reports should focus on risks in the repository content, support files, scripts, workflows, and contribution paths.

Use security reporting for:

- secret exposure in examples
- unsafe instructions
- malicious contribution attempts
- supply-chain risk in scripts or workflows
- prompt-injection risks in included content
- dangerous advice around production systems, real user data, credentials, authentication, payment, CI, deployment, or external tools

Do not open a public issue for a vulnerability report that includes secrets, exploit details, private credentials, or sensitive unpublished details.

If GitHub private vulnerability reporting is enabled for this repository, use it. If it is not enabled, contact the maintainer through the repository owner's preferred GitHub profile contact path, or open a minimal public issue that says a sensitive security report exists without including sensitive details.
