# Start Here

The Vibe Coding Bible is a modular Markdown source package for humans building software with Codex and for AI coaches helping them work safely.

## Recommended human path

1. Read the repository overview:
   [README.md](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/README.md)
2. Use the task index when you know what you are trying to do:
   [indexes/INDEX_BY_TASK.md](https://github.com/frnklfrwsr/vibe-coding-bible/blob/main/indexes/INDEX_BY_TASK.md)
3. Use topic cards for focused guidance:
   [topics/](https://github.com/frnklfrwsr/vibe-coding-bible/tree/main/topics)
4. Use chapter files when you want a broader narrative:
   [chapters/](https://github.com/frnklfrwsr/vibe-coding-bible/tree/main/chapters)

## How to read the source

Prefer the smallest file that answers your question. The repository is designed around modular cards and indexes, not one giant book file.

When a file contains a `VCB:STOP_RETRIEVAL` marker, stop there unless your task explicitly requires related context.

## What this site includes

This first docs-site version provides reader-friendly entry points and navigation pages. It does not mirror every chapter, topic, shortcut, tool card, pricing snapshot, or source register entry.

The GitHub repository remains the canonical source of truth.
